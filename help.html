Software Project Risks

A software project risk some future event, change in customer desires, or choice made by the software project team that has a possibility of occuring and, in the case that it does happen, will cause some sort of negative consequence that hurts the success of the software project.

Learn more about risks

The Condition-Transition-Consequence Format

The CTC format is a sentence structure for a risk that emphasizes the cause-and-effect relationship between a condition and a consequence and makes clear the details of the situations which give the risk a possibility of occurring.

Learn more about the CTC format

Software Engineering Tiny Tools

UM-Dearborn's Software Engineering Tiny Tools are a set of student-designed web apps which each address some unique software development project need. This tiny tool takes a condition/subconditions and a consequence of a risk as input from the user and transforms it into a refined risk sentence which follows the condition-transition-consequence format. Other tiny tools exist which can also help in the management of project risks.
